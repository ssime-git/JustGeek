Prompt Patterns
06 / 21
Good prompts remove degrees of freedom

Weak vs. Strong

WEAK
"Explain embeddings."
  • No audience specified
  • No depth guidance
  • No output format
  • No success criteria
STRONG
"For data scientists, explain embeddings intuitively and mathematically. Use one 2D analogy, one cosine similarity formula, and a 5-line Python pseudo-example."
  • Audience and depth explicit
  • Format is constrained
  • Evaluation target is clear
  • Removes degrees of freedom

Definition
09 / 21
Large Language Model

Two concepts

Language Model

A model trained to estimate patterns in sequences of text. In decoder models, the central objective is next-token prediction.

Large

Massive training corpora, many layers, high-dimensional vectors, and billions of learned parameters.

$$P(x_t \\mid x_{<t})$$
"The chat interface is friendly; the underlying object is a probabilistic sequence model."

Contextual Embeddings
14 / 21
Transformers compute a new vector in context

Contextual representation

The token "bat" is not enough. The model builds a contextual representation by letting surrounding tokens vote on what matters.

Thebatflewcave
Thebathithomerun

Vector Intuition

Embedding vectors place tokens in geometric space. Similar meaning means smaller angle or larger cosine similarity.

In real models this space has thousands of dimensions.

$$\cos(a,b) = \frac{a \cdot b}{\lVert a \rVert \lVert b \rVert}$$
cosine similarity = dot product divided by both vector lengths

Attention is a weighted mixture

For one token, the model scores other tokens, normalizes those scores, then blends information from them.

Not all heads attend to human-interpretable patterns, but this is the useful mental model.

$$\mathrm{Attention}(Q,K,V) = \mathrm{softmax}\left(\frac{QK^T}{\sqrt{d_k}}\right)V$$

Temperature & sampling

Temperature = 0

Deterministic

Always pick the most likely token. No randomness.

Focused - Repetitive - Deterministic

Temperature = 1

Random

Sample from full distribution. More diverse.

Creative - Diverse - Off-topic risk

top-p (nucleus)

Sample from smallest set where cumulative probability exceeds p.

top-k

Only consider the k most likely tokens.

max_tokens

Hard limit on generation length.

Real-world deployment

Risks & trade-offs

Hallucination

Plausible but false content. The model optimizes for coherence, not truth.

Prompt Injection

Malicious inputs that override system instructions.

Privacy

Data sent to APIs may be logged. Consider data residency.

Cost

Priced per million tokens. Input + output both count.

Quantization

Reduce precision (FP16 -> INT8) for speed and size.

RLHF

Human feedback shapes style, not truth. Alignment trade-offs.

Latency

Time to first token vs tokens per second. Different constraints.

Fallbacks

When to refuse, when to escalate, how to handle edge cases.
