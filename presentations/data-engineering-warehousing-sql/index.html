Deux mondes

OLAP et OLTP

Les entreprises ont besoin de données pour le fonctionnement de leurs services. Les services échangent leurs données en effectuant des transactions.

Transactions e-commerce, sinistres en ligne, encodage vidéo : des flux opérationnels à traiter vite.
Enjeux : temps réel, résilience, mise à jour fréquente.

Structured Query
Language

SQL permet de stocker, manipuler et extraire des données dans des bases relationnelles.

SELECT client.nom, COUNT(commande.id) AS nb_cmd FROM client JOIN commande ON client.id = commande.id_client WHERE commande.date >= '2025-01-01' GROUP BY client.nom ORDER BY nb_cmd DESC;
